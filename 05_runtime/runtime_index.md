# Runtime Index  
*Governed execution layer of the Lattice*

The runtime layer defines how the Lattice **executes**, **schedules**, **monitors**, and **reverses** operations.  
It is the living, breathing layer of the system.

This index provides a unified entry point into all runtime documents.

---

## Core Documents

- **runtime_model.yaml** — structural model of runtime  
- **scheduler_spec.yaml** — governed scheduling rules  
- **execution_engine.md** — execution semantics  
- **runtime_envelope.yaml** — runtime boundaries  
- **drift_monitor.yaml** — drift detection and thresholds  
- **reversibility_engine.md** — reversible execution model  
- **runtime_constraints.yaml** — constraints on runtime behavior  
- **runtime_lifecycle.md** — lifecycle of runtime execution  
- **runtime_events.log** — lineage and event log  
- **README.md** — overview of the runtime layer

The runtime ensures:
