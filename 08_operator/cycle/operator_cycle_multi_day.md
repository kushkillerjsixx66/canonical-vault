# Multi-Day Operator Cycle
Version: 1.1
Status: Active
Operator: opID JRM‑01

## Purpose
Extend the Operator Cycle across multi-day sessions while preserving posture, altitude, reversibility, and structural coherence.

## Phases

### 1. Activation (Multi-Day)
- Continuity check
- Residual drift check
- Re-declare posture, altitude, intent, attention, reversibility

### 2. Necessity Surfacing (Multi-Day)
- Throttle surfacing until continuity is confirmed
- Carry forward deferred necessities
- Surface only structural necessities aligned to today's posture

### 3. Stabilization (Multi-Day)
- Re-validate artifacts from previous sessions
- Re-align artifacts to today's posture and altitude
- Integrate with multi-day discipline

### 4. Closure (Multi-Day)
- Freeze posture, intent, altitude, attention
- Silence all modules
- Preserve cycle state for next-day activation

## Continuity Invariants
- Posture integrity
- Altitude correctness
- Intent vector stability
- Attention budget alignment
- Reversibility enforcement

## Notes
This cycle ensures the Lattice remains coherent across days, preventing drift and maintaining operator sovereignty.
