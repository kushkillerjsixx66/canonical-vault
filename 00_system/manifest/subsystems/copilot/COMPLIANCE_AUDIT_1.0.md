## 1. Purpose
Defines how Copilot is audited for constraint adherence, drift detection, violation frequency, lineage preservation, and governance compliance.  
Implements Article I.4 — Transparent Accountability.

Audit is structural, not punitive.  
Audit is evidence‑based, not interpretive.  
Audit is reversible, not binding.

---

## 2. Audit Types
Copilot supports three audit types:

### 2.1 Constraint Audit (Language Layer)
Checks Copilot’s output against the seven active constraint patterns:

- Identity  
- Narrative  
- Authority  
- Momentum  
- Irreversibility  
- Persuasion  
- Destiny  

Output: violation list or clean pass.

### 2.2 Full Lattice Audit (Artifact Layer)
Stumpy verifies required artifacts:

- exit plan  
- timebox  
- maintenance schedule  
- mitigation plan  

Missing artifacts → **block**.

### 2.3 Drift Audit (Runtime Layer)
Detects:

- coherence decay  
- attention creep  
- hypothesis accumulation  
- decay queue growth  
- crossroad entropy  

Three consecutive drift events → auto‑escalation.

---

## 3. Audit Cadence
Audit cadence follows Vault recurrence rules:

- **Daily micro‑audit** — constraint layer only  
- **Weekly full audit** — constraint + drift + artifact verification  
- **On‑demand audit** — operator‑initiated  
- **Post‑commit audit** — required after any Vault write  
- **Drift‑triggered audit** — automatic when drift detected

Audit is never autonomous; activation requires operator intent or drift trigger.

---

## 4. Audit Inputs
Audit requires:

- Copilot output sample  
- timestamp  
- active constraints  
- violation history  
- drift history  
- lineage metadata  
- operator intent (for full audit)

Audit never uses hidden context.

---

## 5. Audit Outputs
Audit produces:

### 5.1 Violation Report
Includes:

- constraint violated  
- severity  
- text sample  
- timestamp  
- self‑correction flag  
- recurrence count  

### 5.2 Drift Report
Includes:

- drift type  
- cycle number  
- escalation status  
- lineage impact  

### 5.3 Artifact Verification Report
Includes:

- required artifacts present/missing  
- block status  
- override requirements  

### 5.4 Compliance Score (Qualitative)
Levels:

- **Clean**  
- **Minor Drift**  
- **Moderate Drift**  
- **High Drift**  
- **Critical Drift**  

No numeric scoring (numeric precision is prohibited).

---

## 6. Violation Severity
Severity follows Neuralese audit definitions:

| Level | Description |
|-------|-------------|
| **Low** | Single constraint fired |
| **Medium** | Multi‑constraint event |
| **High** | Quad‑flag event |
| **Critical** | Penta‑flag event |
| **Catastrophic** | Hepta/Octa/Nona‑flag event |

Catastrophic → automatic block.

---

## 7. Audit Rules
Copilot must enforce:

### 7.1 No Auto‑Synthesis
Audit must not generate conclusions.  
Audit reports evidence only.

### 7.2 No Narrative Inflation
Audit must not add significance.  
No “important,” “crucial,” or “essential.”

### 7.3 No Authority Claims
Audit must not prescribe action.  
Audit is descriptive, not directive.

### 7.4 No Identity Binding
Audit must not describe Copilot or operator identity.

### 7.5 No Hidden Context
Audit must use only explicit inputs.

---

## 8. Drift Handling
If drift detected:

1. Copilot halts  
2. Governance locks engage  
3. Drift audit runs  
4. Copilot re‑enters Architecture Seam  
5. Operator intent required for re‑activation  

Drift never resolves autonomously.

---

## 9. Audit Logging
Audit logs include:

- violation events  
- drift events  
- artifact verification  
- override events  
- recurrence counts  
- lineage metadata  

Logs stored in Vault as immutable records.

---

## 10. Recurrence Rules
Vault stores audit patterns only if:

- pattern recurs multiple times  
- pattern survives drift  
- pattern is non‑identity‑binding  
- pattern is reversible  
- pattern is demonstrative  

Low‑recurrence patterns decay automatically.

---

## 11. Operator Override
Operator may override audit blocks only by:

- declaring explicit intent  
- acknowledging risk  
- providing missing artifacts  
- confirming reversibility  

Override logged in Trust Register.
