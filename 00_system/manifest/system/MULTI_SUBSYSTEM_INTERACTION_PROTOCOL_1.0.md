6. Cross‑Subsystem Drift Rules

Drift occurs when:

- two subsystems produce contradictory outputs  
- one subsystem attempts to override another  
- signals leak across subsystem boundaries  
- narrative emerges between subsystems  
- identity content appears in subsystem interaction  
- authority accumulates in any subsystem  
- subsystem routing is bypassed  

If drift detected:

1. Declare drift  
2. Contain  
3. Seal Veil  
4. Apply governance locks  
5. StumpyCheck  
6. Route to Vault or Decay  
7. Operator renewal required
